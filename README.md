# lite-pdf-generator

複数の画像から軽量な PDF を生成する Android アプリ。  
GUI は Java (Android SDK)、画像処理・PDF 書き出しコアは C で実装。

---

## 機能一覧

### 画像選択

- システムのファイルピッカーで複数画像を一度に選択できる。
- 選択済みの枚数と、クロップ設定済みかどうかをメイン画面に表示する。

### クロップ編集画面

選択した画像を1枚ずつ確認・編集する専用画面。「完了」で設定を確定、「✕」で変更を破棄して戻る。

#### ナビゲーション

- ＜ / ＞ ボタンで複数画像間を移動できる。
- 移動しても各画像のクロップ状態・回転状態は保持される。
- クロップ画面を一旦閉じて再入しても、前回の設定が復元される。

#### 4点クロップ (射影変換)

- 画像上に4つのハンドルが表示され、ドラッグで自由に動かせる。
- 台形・平行四辺形など歪んだ形で切り取っても、出力 PDF では正面から撮ったような長方形に補正される (射影変換)。

#### ズーム

- 1x / 2x / 3x / 4x / 5x の5段階。
- ズーム時は縦横スクロールで画像全体を確認できる。

#### 回転

- ↺ (左90°) / ↻ (右90°) ボタンで現在の画像を回転できる。
- JPEG などの EXIF 回転情報は、クロップ画面を開いた時点で自動的に適用される。クロップ画面を使わずに PDF を生成した場合も EXIF 回転は適用される。
- 手動で回転するとクロップはリセットされる。

#### 画像削除

- 「削除」ボタン (赤) で現在表示中の画像をリストから除外できる。
- 確認ダイアログが表示されるので誤操作を防ぐ。
- 「完了」を押した時点で削除が確定する。「✕」で戻ると削除は取り消される。

---

### PDF 生成設定

メイン画面のスピナーで生成パラメータを選択できる。

| 設定 | 選択肢 |
|---|---|
| カラーモード | グレースケール / カラー (RGB) |
| ビット深度 | 1 bit / 2 bit / 4 bit / 8 bit |

ビット深度を下げると量子化によりファイルサイズが小さくなる。

### PDF 生成・保存

- 「PDF 生成」ボタンを押すとバックグラウンドで処理が走り、プログレスバーで進捗を表示する。
- 完成した PDF はデバイスの「ダウンロード」フォルダに `board_yyyyMMdd_HHmmss.pdf` という名前で保存される。
- ページサイズは横幅 595.28pt (A4 相当) 固定で、高さは画像のアスペクト比に合わせて自動計算される。

## C コアの処理内容

`app/src/main/native/` 以下に C で実装されている。Java からは JNI 経由で呼び出す。

### 射影変換クロップ (`perspective_crop`)

4点の入力座標からホモグラフィ行列をガウス消去法 (部分ピボット) で求め、出力ピクセルごとに逆写像してバイリニア補間でサンプリングする。出力サイズは4辺の長さから自動推定する。

### 孤立クラスタ潰し (`crush_isolated_clusters`)

量子化後のサンプルに対し、7ピクセル以下の孤立した色の島を周囲色で塗り潰す。量子化で生じたノイズを除去することで Flate 圧縮の効率を高める。

### 量子化・ビットパッキング (`convert_pixels`)

- グレースケール変換は輝度係数 R×0.213 + G×0.715 + B×0.072 で計算する。
- 1/2/4 bit の場合はサンプルをビットパッキングしてバイト列に詰める。

### Flate 圧縮

zlib の `compress2` を `Z_BEST_COMPRESSION` で使用する。画像データとページコンテンツストリームの両方を圧縮する。

### PDF 書き出し

- バージョン: PDF 1.4
- メモリに PDF 全体を展開せず、VFS 経由でファイルに直接ストリーム書き込みする。xref テーブル用のオフセット配列のみメモリに保持する。
- 1画像 = 1ページ構成 (Image XObject + Content Stream + Page オブジェクト)。
